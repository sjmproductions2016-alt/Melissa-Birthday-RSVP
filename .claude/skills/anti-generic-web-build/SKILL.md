# Anti-Generic Web Build Skill

## Mission

Prevent generic AI-looking website output.

The site must not merely work. It must have a clear visual thesis, custom art direction, strong hierarchy, purposeful motion, responsive polish, and prompt-specific execution.

## Generic AI Risk Scan

Before coding, inspect the current site or implementation plan and identify whether it contains:

1. Centered hero with vague subtitle and generic CTA
2. Gradient background with no custom visual system
3. Repeated rounded cards with identical rhythm
4. Random floating icons or emojis
5. Stock-looking section sequence
6. Generic image cards with weak purpose
7. Bland typography with no hierarchy
8. Motion that only decorates
9. Glassmorphism without narrative purpose
10. Same padding, card shape, and layout rhythm across every section
11. No signature hero mechanism
12. No memorable section interaction
13. No page-specific visual metaphor
14. Placeholder-feeling copy
15. Design that could fit any event by changing the name

If 3 or more are present, mark the design as GENERIC-RISK.

## Melissa Birthday RSVP Visual Thesis

A cinematic princess brick kingdom RSVP experience: a purple-pink-gold royal party world built from glossy toy-brick structures, crown details, magical cake moments, and polished family-event photography.

## Required Upgrade Moves

When improving a generic build, apply at least 7 of these:

1. Replace generic hero with a signature hero composition.
2. Add a custom visual metaphor tied to the event.
3. Vary section layouts instead of repeating cards.
4. Add one memorable interactive section.
5. Use stronger typography hierarchy.
6. Create intentional image treatment.
7. Add art-directed background layers.
8. Add purposeful entrance or scroll effects.
9. Create custom CTA treatment.
10. Improve form design and confirmation state.
11. Use asymmetric or editorial layout where appropriate.
12. Add a distinctive navigation/section rhythm.
13. Use design tokens for color, spacing, shadows, borders.
14. Remove placeholder-feeling copy.
15. Make mobile feel designed, not stacked.

## Output Before Coding

Return:

1. Generic-AI Risk Score from 1-10
2. What currently feels generic
3. Chosen visual thesis
4. Three design directions
5. Recommended direction
6. Specific files to change
7. Upgrade plan
8. Ask for approval before coding

## Final Audit

After coding, return:

1. What generic patterns were removed
2. What custom visual system was added
3. Which sections now feel signature
4. Mobile polish check
5. Accessibility check
6. Build result
7. Final verdict: SHIP / REVISE / BLOCK
